## 2. 顧客ジャーニー（概要）

```
認知（Instagram・口コミ・通りがかり）
  → 興味（メニュー・価格・営業時間の確認）※ここで Web / LINE が効く
  → 来店・購入
  → 満足・シェア（UGC）
  → リピート（LINE・限定情報）
```

**現状のボトルネック想定**

- メニュー・営業時間・場所が Instagram 以外でまとまっておらず、**検索コストが高い**
- リピート導線が **フォロー頼み** で、到達率に依存

→ **Web（信頼・SEO・一覧性）** と **LINE（到達・クーポン）** で補完するのが優先度高。

## 4. Web「お知らせ／ニュース」欄のノーコード更新（ツール比較）

トップのニュース欄（`03-website-project.md` のセクション案）を、**スタッフが Git やコードを触らず**更新する前提での選択肢です。いずれも **最初の API 接続・型定義・デプロイ**は実装者側の作業が必要になることが多いです。

### 4-1. 比較の観点

| 観点 | 説明 |
|------|------|
| 編集体験 | ブラウザのフォーム・表・ブロックエディタなど |
| 公開の流れ | 即時反映（ISR）／再ビルド後反映／別サイトへリンクのみ 等 |
| コスト | 無料枠の有無・ホスティング別途 |
| 向く規模 | 月数本のお知らせ〜週数本の更新 |

### 4-2. ノーコード寄りの手段（複数案）

| 手段 | 編集者の操作 | 初期セットアップ（担当の目安） | 月額の目安 | メリット | 注意点 |
|------|----------------|----------------------------------|------------|----------|--------|
| **ヘッドレス CMS**（[microCMS](https://microcms.io/) / [Newt](https://www.newt.so/) / [Contentful](https://www.contentful.com/) 等） | 日本語 UI の管理画面で「日付・タイトル・タグ・本文」を入力し公開 | API キー発行、Next.js から取得、再ビルド or ISR の設定 | 無料枠〜（プランによる） | 項目追加が柔軟、下書き・権限・プレビューが取りやすい | 実装と契約の初期設計が必要 |
| **Decap CMS**（[Decap CMS](https://decapcms.org/)、旧 Netlify CMS） | ブラウザのフォームで入力 → **Git リポジトリにコミット** | GitHub 等との OAuth、YAML に「お知らせ」コレクション定義、ホスティングと連携 | **ほぼ無料**（ホスティングのみ） | ランニングコストが低い、データは自前 Git に残る | 編集者に「プレビュー → 公開」の流れのマニュアルが必要（`03` にも記載あり） |
| **WordPress**（[WordPress.com](https://wordpress.com/) のブログ、または自店サーバの WP） | 投稿画面のみ使う運用に絞る | サブドメイン `news.◯◯` にブログだけ置く／トップは一覧 API か埋め込みで取り込み | .com はプラン次第／自前はサーバ代 | スタッフに **WP 慣れ**があると学習コスト最小 | 本体・プラグイン更新、セキュリティ対策の運用が別途必要 |
| **Notion**（[Notion](https://www.notion.so/) のデータベースを公開 DB に） | Notion で行追加・下書き | Notion API でビルド時取得、または埋め込み／再生成パイプライン | 無料〜（Plus 等） | 社内メモと一体で回せる、非エンジニアが即慣れる | **SEO・表示速度・キャッシュ**は設計次第。本番利用は実装者と方針確認推奨 |
| **Google スプレッドシート** | 表の1行＝1件のお知らせとして追記 | 公開用シート or API + ビルド時 fetch、列はテンプレ固定 | 無料 | 表計算が得意な人が担当しやすい | 誤公開・列ずれ防止に **権限とテンプレ固定**が必須 |
| **Airtable**（[Airtable](https://www.airtable.com/)） | 表＋添付・タグのビジュアル管理 | Base の API、サイト側はビルド時 or サーバレスで取得 | 無料枠〜 | 画像・タグ・担当者列などを視覚的に管理しやすい | API レート制限、無料枠の行数上限 |

### 4-3. 選定の早見

- **コスト最優先・月数本の更新** → **Decap CMS** または **microCMS 無料枠**から検討。
- **日本語の管理画面と下書き・権限を重視** → **microCMS** / **Newt**。
- **店に WordPress 運用の慣れがある** → **WP の「投稿」だけ**をニュース専用にする案。
- **とにかく今日から社内で下書きしたい**（プロトタイプ）→ **Notion**（本番の取り込み方法は実装者と合意）。

### 4-4. 運用ルール（共通）

- 公開前に **責任者確認**（休業・価格・誤字）。
- 可能なら **お知らせ1件＝1 URL**（SNS・LINE からのリンクで迷子を減らす）。
- 景表・薬機に触れそうな表現は `01-brand-strategy.md` のガードレールに合わせる。

## 5. 施策ロードマップ（提案）

### フェーズ 0（即時・低コスト）

- Instagram プロフィールに **リンク集**（メニュー PDF または簡易 LP）を置く暫定対応
- ハイライトで **メニュー / アクセス / よくある質問** を整理

### フェーズ 1（1〜2 ヶ月）

- **Web サイト MVP** 公開（`03-website-project.md`）
- **LINE 公式** 開設・リッチメニュー・ウェルカムメッセージ（`04-line-official-project.md`）
- 店頭に **LINE 友だち追加 QR** と **一言 CTA**（例: 「今だけクーポン」は実施する場合のみ）

### フェーズ 2（2〜4 ヶ月）

- 月 1〜2 回の **LINE 配信**（新メニュー・定休・イベント）
- Instagram と **同じビジュアルルール** で Web・LINE 画像を統一
- 近隣 **1〜2 店舗との相互送客** やコラボの検討（ジム・ヨガ・サロン等）

### フェーズ 3（中長期）

- 季節キャンペーンの型化（春いちご、夏マンゴー等）
- 必要に応じて **Meta 広告** または **インフルエンサー**（地域マイクロ優先）
