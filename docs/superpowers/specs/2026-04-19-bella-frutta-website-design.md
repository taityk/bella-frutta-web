## 5. データ管理（Google Sheets）

### シート構成

**`menu` シート**（1〜2週間に1回更新）

| 列 | 内容 |
|----|------|
| id | 連番 |
| category | smoothie / fruit / other |
| name_ja | 商品名（日本語） |
| name_en | 商品名（英語） |
| price | 価格（税込、数値） |
| description_ja | 説明文（日本語） |
| description_en | 説明文（英語） |
| image_url | 画像URL |
| active | TRUE / FALSE（非表示フラグ） |

**`news` シート**（フェーズB追加）

| 列 | 内容 |
|----|------|
| id | 連番 |
| title_ja | タイトル（日本語） |
| title_en | タイトル（英語） |
| body_ja | 本文（日本語） |
| body_en | 本文（英語） |
| published_at | 公開日 |
| active | TRUE / FALSE |

### 更新フロー

```
Google Sheets を編集（スタッフ）
　↓
Netlify Build Hook をトリガー（Webhook URL をシートに貼り付けてボタン化）
　↓
自動ビルド・デプロイ（約5〜10分）
　↓
サイトに反映
```

## 6. 多言語対応

- **方針**: フェーズAから組み込む（後回しにすると大規模リファクタが必要）
- **ライブラリ**: `next-intl`
- **URL構造**: `/`（日本語デフォルト）、`/en/`（英語）
- **テキスト管理**: `messages/ja.json` / `messages/en.json`
- **英語テキスト**: 実装者が日本語から翻訳。公開時はプレースホルダーでも可、後から更新

### UI方針

- **レスポンシブ**: モバイルファースト（Instagram流入はスマホ主）
- **余白**: 広め。写真を「置く」感覚
- **角丸**: カード `1.25rem`、大モジュール `1.5rem`
- **アニメーション**: 控えめなフェードアップのみ
- **電話番号**: 非掲載。すべての問い合わせをLINEに誘導

### 写真素材

| 種別 | 状況 |
|------|------|
| 商品（スムージー・フルーツ） | あり |
| 制作工程 | あり |
| 店長 | あり |
| 店内・外観 | なし（使用しない） |

## 9. 法人ページ（`/corporate`）

- **提供サービス**: スムージー卸（B2Cと同等商品）、フルーツギフト
